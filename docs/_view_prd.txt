=== PARAGRAPHS ===
1. Three-Tier Viewing Architecture
1.1 Tier Overview
Artifacts are surfaced at three increasing levels of detail. The user moves down the hierarchy as their attention deepens: from glancing at the Journey tab, to expanding a layer row for context, to opening a dedicated page for editing.
1.2 Tier Transitions
Glance to Preview: click anywhere on the Glance card. Smooth height transition (150ms). Other Preview cards in the same layer row collapse — only one Preview open per layer at a time.
Preview to Full View: click 'Open full artifact'. Browser navigation to /simulations/:id/artifacts/:artifact_id. The GCC scroll position and Preview state are stored in sessionStorage.
Full View back to GCC: click 'Back to your simulation' in the Full View header. Restores GCC scroll position and re-opens the Preview that was open.
2. Tier 1 — Glance Card
2.1 Layout
The Glance card sits inline within the GCC layer row alongside the action description from SIM-PRD-GCC-002. Approximately 84 pixels tall, full row width, with a left accent border in the layer's color (purple for L1, teal for L2).
2.2 Multiple Artifacts Per Layer Row
As cycles accumulate, a layer row can have many artifacts. The GCC presents only the most current artifact per action_type by default. A 'Show prior artifacts (N)' link at the bottom of the layer row reveals archived artifacts grouped by action_type.
Default view: latest artifact_version per action_type, chronological by created_at desc.
Archived view: collapsed by action_type. Click action_type group to expand list of past artifacts.
Limit: never more than 8 Glance cards in default view. If there are more, the bottom card becomes 'Show 4 more' which expands inline.
3. Tier 2 — Preview
3.1 The Artifact-Aware Preview
The Preview is what makes the GCC feel like a real wealth dashboard rather than a list of completed tasks. Instead of generic JSON, each artifact type renders in its natural visual format. The user sees a rate card as a rate card. They see a cold email as a sequence.
The implementation is a Jinja2 template selected by artifact.action_type. Each template renders artifact.payload (JSON) into the appropriate visual layout.
3.2 Preview Common Structure
4. Per-Artifact Preview Formats — Layer 1
Each Layer 1 artifact type has a dedicated Preview template. The template renders artifact.payload into the visual format that matches what the artifact represents.
5. Per-Artifact Preview Formats — Layer 2
6. Tier 3 — Full View Page
6.1 Page Layout
The Full View page at /simulations/:id/artifacts/:artifact_id is where the user does deep work on an artifact. The page is structured as a focused single-column workspace — no GCC layer rows, no chat panel — to encourage uninterrupted reading and editing.
6.2 Inline Editing
Every editable field is click-to-edit. There is no separate 'Edit mode' toggle. The user clicks any field and it becomes editable. This pattern matches modern document editing (Notion, Linear).
6.3 Save as New Version
Clicking 'Save as new version' creates a new artifact_versions record with the edited payload. The previous version is preserved with is_current=0. The new version becomes current with version_number incremented.
7. Version History — Count Badge with Click-Through
7.1 Default View — Current Version Only
The Full View page defaults to showing the current version only. The version count badge in the header — 'v3 · 2 prior' — is the only indicator that prior versions exist. Most users never click it. The default UX is clean and uncluttered.
7.2 Version History Sidebar
Clicking the version count badge slides in a right-side panel (320px wide) listing all versions of this artifact in reverse chronological order.
7.3 Auto-Generated Edit Summaries
Each version stores an edit_summary that describes the changes — short text generated automatically when a version is saved. This avoids the engineering complexity of full structured JSON diffs while giving users meaningful context.
8. Data Model — Extensions to artifact_versions
8.1 Schema Additions
The existing artifact_versions table is extended with three new columns to support the editing and versioning UX. No new tables are introduced.
8.2 Auto-Save Draft Mechanism
While the user is editing, a separate draft is persisted every 30 seconds without creating new versions. Drafts use a sessionStorage key on the client and a corresponding server-side cache (Redis) keyed by artifact_id and user_id.

10. Functional Requirements
FR-VIEW-01: Every Layer 1 and Layer 2 agent-generated artifact is viewable through three tiers: Glance card in the GCC Journey tab layer row, Preview on inline expand within the layer row, Full View at /simulations/:id/artifacts/:artifact_id. Users move between tiers with single-click navigation.
FR-VIEW-02: The Glance card displays artifact icon, plain-English artifact name (never the action_type string), status badge (Draft / Scheduled / Deployed / Live / Archived), one-line summary, primary action button with status-aware label, and version count badge if version_number > 1.
FR-VIEW-03: A layer row displays only the latest artifact_version per action_type by default. Prior artifacts are accessible via 'Show prior artifacts' link grouped by action_type. The default view is capped at 8 Glance cards per layer row with 'Show N more' expansion.
FR-VIEW-04: The Preview tier renders artifacts in artifact-aware visual formats per Sections 4 and 5. Each of the 19 L1 and L2 agent action_types has a dedicated Jinja2 preview template that renders artifact.payload into the appropriate visual layout. Only one Preview is open per layer row at a time.
FR-VIEW-05: The Preview action bar provides three buttons: 'Open full artifact' (primary), 'Edit' (secondary), 'Re-run' (secondary). Plus contextual deploy buttons based on artifact type and current status.
FR-VIEW-06: The Full View page renders the artifact in a focused single-column workspace at max-width 760px. Header is sticky and shows: Back link, artifact name, layer label, version count badge, status badge, primary action button. Action bar is sticky at bottom with Save / Discard / Re-run / Archive / Download.
FR-VIEW-07: All editable fields in Full View are inline click-to-edit. There is no separate Edit mode toggle. Field types include plain text, multi-line text, currency, number, date, list (with add/delete/reorder), URL, and rich-text email body. Read-only computed fields render in muted color.
FR-VIEW-08: Edits are auto-saved as drafts to Redis every 30 seconds while editing. Drafts persist for 24 hours and do not create new versions. Full View page checks for an existing draft on load and offers to restore unsaved changes.
FR-VIEW-09: Clicking 'Save as new version' creates a new artifact_versions record with the edited payload, increments version_number, sets is_current=1 on new version and is_current=0 on prior, sets edited_by='user', auto-generates edit_summary, sets parent_version_id, and clears the draft.
FR-VIEW-10: Version history is exposed via a count badge in Full View header showing 'v[N] · [N-1] prior'. Clicking opens a 320px right-side panel listing all versions in reverse chronological order. Each row shows version number, edited-by indicator, relative timestamp, and edit_summary.
FR-VIEW-11: From the version history sidebar, clicking 'View' on a prior version loads that version's payload into Full View body in read-only mode with a banner indicating non-current state. Clicking 'Restore' creates a new artifact_version with the prior version's payload.
FR-VIEW-12: Auto-generated edit_summary follows fixed format rules: 'Generated by [agent_name] in cycle [N]' for agent versions, 'You changed [field name]' for single-field user edits, 'You edited [N] fields' for multi-field user edits, 'Restored from version [N]' for restorations, 'Co-pilot updated [field name] at your request' for chat co-pilot edits.
FR-VIEW-13: The Full View page is reachable from: GCC Preview 'Open full artifact' button, chat co-pilot artifact references (clickable links), notification CTAs (notification_type='agent_complete'), direct URL navigation.
FR-VIEW-14: Re-run dispatches the originating agent action_type with optional custom_prefill_inputs. On completion a new artifact_version is created with edited_by='agent'. The agent action's metadata embedding (per SIM-ENG-MODEL-001) is preserved.
FR-VIEW-15: Archive sets the artifact status to 'archived' and removes it from the GCC layer row default view. Archived artifacts remain accessible via 'Show prior artifacts' and via direct URL. Archive is reversible — a 'Restore from archive' link in Full View restores the artifact to default view.
